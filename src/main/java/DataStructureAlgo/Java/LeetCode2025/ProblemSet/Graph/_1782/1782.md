### 1782 — Count Pairs of Nodes (Interview Summary)

**Core Goal**

For each query `q`, count **node pairs `(u,v)`** such that:
```
deg[u] + deg[v] > q
```

But **shared edges between `(u,v)` inflate the degree sum**, so we must **correct those pairs**.

---

# Mental Model (Remember This)

Think of the problem as **3 stages**.

---

## 1️⃣ Collapse Graph → Degree Array

We don't care about full graph traversal.

Compute:

```text
degree[i] = number of edges connected to node i
```

Also store:

```text
pair_freq[(u,v)] = number of edges between u and v
```

Purpose:

* `degree` → used for **pair counting**
* `pair_freq` → used for **fixing duplicates**

---

## 2️⃣ Count Valid Pairs Using Two Pointers

Sort degrees.

Then count pairs where:

```
deg[i] + deg[j] > q
```

Using **two pointers**.

This counts **all pairs in O(N)** per query.

BUT this step **does not know about shared edges**.

---

## 3️⃣ Fix Overcounted Pairs

For every pair `(u,v)` that has `c` shared edges:

Check:

```
deg[u] + deg[v] > q
```

but

```
deg[u] + deg[v] - c \le q
```

If true → this pair **was falsely counted**.

So:

```
ans -= 1
```

Because `(u,v)` is **one node pair**, regardless of how many edges connect them.

---

# Key Insight (Interview Gold)

Two-pointer counts using **degree only**.

But **shared edges artificially inflate degree sum**.

So we **subtract exactly the wrongly counted pairs** afterward.

---

# Complexity
E = len(edges), N = no. of nodes, Q = len(queries) 
1. Compute degree + shared_freq_map : O(E)
2. Sorting degree : O(N.logN)
3. Finding incidents (two_sum) : O (N) 
4. Fixing incidents: O(E) ; as freq_map can have at most E edges.
5. Per query : O(Q * (N+E))

Time:

```
O(E + N.log N + QN)
```

Space:

```
O(E + N)
```

---

# One Sentence Memory Trick

> Convert graph → **degree array**,
> count pairs with **two pointers**,
> then **fix inflated pairs using shared-edge map**.

---

# Pattern Tag (Very Important)

This is a **Hybrid Problem**:

```
Graph Degree Analysis
+ Two Pointer Pair Counting
+ Post Correction using HashMap
```
